// 1. 基本的sql语句？ * 1

// 2. 分库分表是如何做的? * 1
    分库将数据拆分到多个数据库实例中，以减轻单个数据库的负担，通常通过水平或垂直策略实现。
    分表将一个大表拆分成多个小表，通过水平或垂直拆分来提高性能和管理性。分库分表需要解决数据一致性、复杂查询、管理维护等挑战。
        
// 3. 布隆过滤器的原理？ * 1
    布隆过滤器是一种空间效率高的概率型数据结构（位数组+哈希函数），快速判断元素是否在集合中。
    a.添加元素：计算元素的哈希值，并将对应的位设置为1。
    b.查询元素：计算元素的哈希值，检查对应的位是否为1。可能假阳，但一定真阴。
        
// 4. 非常大的文件，只有1G内存，如何统计大文件中元素的出现次数？ * 1
    使用位图（bitmap）可以帮助处理大文件中的元素出现次数，特别是当元素范围有限时。
    32位无符号整数的范围是0到2^32-1，需要 4,294,967,296 位。每个位需要1比特，所以总空间为4,294,967,296比特。
    转换为字节：4,294,967,296 / 8 = 536,870,912字节。转换为 MB：536,870,912 / (1024 * 1024) ≈ 512MB < 1GB。


// 5. 查询到了B+树的叶子节点，之后的查找是如何做的？ * 1

// 7. MyBatis 中 #和$有什么区别？ * 1
    使用#是防止SQL注入的推荐方式，因为它能够自动处理参数的转义。
    例如：SELECT * FROM users WHERE username = '#{username}'，#{username} 是一个参数占位符。
    使用$不会将用户输入直接插入到SQL语句中，需要对参数值进行严格的验证和清理，以避免SQL注入攻击。
    例如：SELECT * FROM users WHERE username = '${username}'，${username} 为 ' OR '1'='1 会导致SQL注入攻击。


// 8. 本地缓存和Redis缓存的区别？ * 1
    本地缓存：位于本地应用的内存，重启后丢失数据。单机应用缓存数据。速度非常快，直接访问内存。容量受限于应用的内存。
    Redis 缓存：位于服务器的内存，重启后可恢复数据。多机多应用共享缓存。速度快，通过网络但优化。容量受限于服务器配置。


// 9. 如果有一个接口存的是大Key，QPS比较低，另外有10个接口，QPS非常高，会有什么影响？ * 1
    高QPS接口会与大Key操作争夺CPU、内存和网络资源。大Key成为性能的主要限制因素，影响整体系统的稳定性和响应时间。


// 10. 你平常是怎么使用RDB和AOF的？ * 1
    RDB：当对数据一致性的要求不是特别严格，允许数据丢失时，RDB 是一个较好的选择。
    配置：通过 redis.conf 设置。例如：SAVE 900 1，900秒内至少有1个写操作则保存快照。
    AOF：适合对数据持久化要求高的场景，尤其是需要尽可能减少数据丢失的场合。例如，关键业务数据存储。
    配置：通过 redis.conf 设置。例如：appendonly yes，appendfsync everysec，每秒同步一次 AOF 文件。


// 11. 如何保证Redis缓存和MySQL数据库的一致性？ * 4
    延迟双删 + 消息队列（处理重启等失效情况）。例如：
    线程A：删除缓存数据，还没更新数据库数据。
    线程B：查询缓存失败，查询数据库得到旧值，更新缓存数据。
    线程A：更新数据库数据，最后删除缓存数据。
        
// 12. 分表分表是什么？依据是什么？ * 2
    分库：将数据分散到多个数据库中，以减轻单个数据库的负担。例如，按业务模块或数据量将数据分到不同的数据库中。
    分表：将一个大的表拆分为多个表，以提高查询性能和并发处理能力。分为水平分表、垂直分表。
    依据：数据量过大、访问压力过大，业务需求、完善扩展性。

// 18. 登录的信息是存储在前端的什么位置？cookie和session有什么区别？ * 1
    登录信息一般不应存储在前端，以确保安全性。前端常见的存储方式是 cookie。
    cookie：客户端存储，容量小。存储少量的数据，比如用户偏好。每次请求时，浏览器会自动将相关的 cookie 发送到服务器。
    session：服务器存储，容量大。存储复杂的数据，比如用户信息。使用唯一的 session ID（存储在cookie中）标识用户的会话。
        
// 19. ==和equals的区别？为什么重写equals方法？为什么重写hashcode方法？ * 2
    == 用于比较对象的引用地址，equals() 用于比较对象的引用或内容。
    a.对于自定义类，如果不重写 equals()，则会错误的比较对象的内容。
    b.对于使用哈希表的类，如果不重写 hashCode()，则不同对象可能会有相同哈希值（哈希碰撞）。


// 20. Spring 中的 IOC 和 DI？ * 1

    b.依赖注入（DI）：通过构造函数、setter 方法或字段注入依赖，实现组件之间的解耦。
    例如：将支付接口的实现类注入到订单接口。更改支付接口的实现，而无需修改订单接口。

    b.Spring Boot 旨在简化 Spring 的开发过程。它通过提供自动配置和内嵌 Tomcat 服务器来简化项目设置。


// 22. JWT令牌结构由什么组成的？JWT令牌中载荷部分是安全的吗？ * 2
    a.头部（Header）：通常包含令牌的类型和所使用的签名算法。
    b.载荷（Payload）：包含声明，可以是注册声明、公共声明、私有声明。
    c.签名（Signature）：用于验证令牌的真实性和完整性。
    JWT 的载荷部分不是加密的，只是编码后的明文，因此敏感信息不应存储在载荷中。


// 23. 消息中间件了解过吗？消费模式有几种？ * 1
    a.单一消费者模式：一个消息队列由一个消费者进行消费，消息按照顺序逐个被处理。
    b.竞争消费者模式：一个消息队列由多个消费者进行消费，消息被多个消费者竞争处理。
    c.发布订阅模式：消息被发布到一个交换机，然后广播到所有队列，多个消费者从这些队列中消费消息。
    d.路由模式：消息被发布到一个交换机，然后路由到特定队列，多个消费者从这些队列中消费消息。

// 1. 无重复字符的最长子串（滑动窗口） * 1


class Solution {
    public int lengthOfLongestSubstring(String s) {
        if (s.length() == 0) return 0;
        int res = 0, left = 0;
        HashMap<Character, Integer> map = new HashMap<>();
        
        for (int i = 0; i < s.length(); i++) {
            Character c = s.charAt(i);
            if (map.containsKey(c)) {
                left = Math.max(left, map.get(c) + 1);
            }
            map.put(c, i);
            res = Math.max(res, i - left + 1);
        }
        return res;
    }
